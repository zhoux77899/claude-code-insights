# Plugins Data Directory

This directory stores generated datasets and reports for Claude Code plugin repositories.

## Files in This Directory

| File | Purpose |
|------|---------|
| `repos.json` | Latest ranked repository dataset (full repository objects from GitHub API) |
| `history.json` | Rolling time-series snapshots (stars/watchers/forks by date) |
| `plugins-daily-insight.md` | Human-readable daily report |
| `cached-repos.txt` | Cached list of repository full names for expanded coverage |
| `README.md` | This documentation |

## Data Source

Repositories are discovered via GitHub code search for:

```text
filename:marketplace.json path:.claude-plugin
```

API query used by scripts:

```text
https://api.github.com/search/code?q=filename:marketplace.json+path:.claude-plugin
```

## Generation Workflow

Generated by `.github/workflows/plugins-daily-insight.yml` (daily at `00:00 UTC` and manual trigger supported).

Pipeline:

1. `scripts/fetch_repos.sh` fetches paged search responses.
2. `scripts/dedupe_repos.py` deduplicates repositories by `full_name`.
3. `scripts/sort_repos.py` fetches repository metadata and sorts by stars.
4. `scripts/chronicle_repos.py` updates historical metrics.
5. Workflow writes artifacts under `plugins/`.

## `repos.json` Schema

Top-level shape:

```json
{
  "total_count": 0,
  "last_updated_time": "YYYY-MM-DD HH:MM:SS",
  "items": [
    {
      "id": 123,
      "name": "example-repo",
      "full_name": "owner/example-repo",
      "html_url": "https://github.com/owner/example-repo",
      "description": "Repository description",
      "stargazers_count": 0,
      "watchers_count": 0,
      "forks_count": 0,
      "pushed_at": "2026-01-01T00:00:00Z"
    }
  ]
}
```

Notes:

- `items` are repository objects returned by GitHub repository API (`/repos/{owner}/{repo}`), so additional fields may exist.
- `total_count` and rankings change over time; treat them as dynamic values.

## `history.json` Schema

Shape:

```json
{
  "owner/example-repo": {
    "2026-01-01": {
      "stargazers_count": 100,
      "watchers_count": 100,
      "forks_count": 10
    },
    "2026-01-02": {
      "stargazers_count": 102,
      "watchers_count": 102,
      "forks_count": 11
    }
  }
}
```

Notes:

- Key is repository `full_name`.
- Each date key stores one daily snapshot.
- Default retention is 30 days (configurable through `chronicle_repos.py --keeping-days`).

## `plugins-daily-insight.md`

Daily Markdown report containing:

- Generation timestamp and indexed repository count.
- Top 30 repositories by stars.
- Up to 15 repositories active in the last 7 days.

The report is designed for quick reading, while `repos.json` and `history.json` are intended for programmatic consumption.

## Usage Examples

### Read top 10 repositories (Python)

```python
import json

with open("plugins/repos.json", "r", encoding="utf-8") as f:
    data = json.load(f)

top10 = data["items"][:10]
for repo in top10:
    print(repo["full_name"], repo["stargazers_count"])
```

### Read one repository history (JavaScript)

```javascript
import fs from "node:fs";

const history = JSON.parse(fs.readFileSync("plugins/history.json", "utf-8"));
const repo = "owner/example-repo";
console.log(history[repo] || {});
```
